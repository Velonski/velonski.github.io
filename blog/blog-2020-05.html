  • Easy2Boot: утилита для создания мультизагрузочных дисков

    04.05.2020

    Easy2Boot - это сейчас лучшее, на мой взгляд, программное обеспечение, чтобы создать загрузочную флешку и "запихнуть" туда десяток дистрибутивов разных операционных систем.

    Принцип работы:

    • программа особым образом форматирует флешку
    • на флешке появляется специальный пустой каталог
    • в этом каталоге размещаем файлы *.iso операционных систем
    • загружаем компьютер, выбрав флешку в качестве загрузочного диска
    • выбираем файл *.iso, с которого хотим загрузить операционную систему
    • ждём загрузки

    Если хотим поменять состав дистрибутивов на флешке, то просто удаляем файлы *.iso и заменяем их другими.
    Easy2Boot также может использовать файлы *.ima, *.img, *.vhd, *.vhdx, *.wim.

    Подробно можно почитать на сайте проекта: https://www.easy2boot.com

    Я сохранил для себя дистрибутив программы и небольшие инструкции к ней.

    Cсылки на каталог в облачном хранилище:

    • Яндекс.Диск
    • Google Drive

    Файлы в каталоге:
    • Easy2Boot_v2.02.exe - файл для установки Easy2Boot в ОС Windows 7/8/10
    • Easy2Boot_v2.02_password_is_e2b.zip - просто программа в ZIP-архиве, которую можно запускать без установки в ОС Windows XP. Также содержит скрипты для запуска Easy2Boot в Linux
    • easy2boot-make-an-easy2boot-usb-drive.pdf - инструкция с офиц. сайта по созданию загрузочной флешки в среде Windows
    • easy2boot-make-usb-using-linux.pdf - инструкция по созданию загр. флешки в среде Linux

  • Earlyoom: предотвращение нехватки оперативной памяти в Linux

    23.05.2020

    Программа Earlyoom нужна для ОС Linux. Она следит за количеством доступной памяти и принудительно закрывает приложения, если памяти осталось совсем мало. Это очень актуально, когда надо работать на компьютере с небольшим объёмом памяти.

    Earlyoom запускается и работает в фоновом режиме из командной строки.

    Я пользуюсь программой так: 

    
earlyoom -m 2 -r 60 --avoid '(^|/)(init|Xorg|ssh)$' --prefer '(^|/)(firefox|firefox-esr|java|chromium|mozilla|seamonkey)$' 1> /dev/null 2> /dev/null &
    Эта команда означает:
    • "убивать" лишние процессы в системе, когда объём доступной памяти уменьшается ниже 2% от максимума
    • выполнять проверку на доступную память каждые 60 секунд
    • ни в коем случае не "убивать" программы init, Xorg, ssh
    • в первую очередь "убивать" процессы, наименование которых соответствует регулярному выражению '(^|/)(firefox|firefox-esr|java|chromium|mozilla|seamonkey)$'. В данном примере, прежде всего убиваться будут браузеры Firefox, Seamonkey, Chromium, а также Java-приложения

    В некоторых дистрибутивах Linux программа Earlyoom уже установлена по умолчанию и работает в фоновом режиме при загрузке системы. В этом случае её настройки хранятся в файле /etc/default/earlyoom.

    У меня этот файл имеет такое содержание: 

    
EARLYOOM_ARGS="-m 2 -r 60 --avoid '(^|/)(init|Xorg|ssh)$' --prefer '(^|/)(firefox|firefox-esr|java|chromium|mozilla|seamonkey)$'"
